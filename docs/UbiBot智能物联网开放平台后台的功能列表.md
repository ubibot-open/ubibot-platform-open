# UbiBot智能物联网开放平台后台的功能列表

本清单按物联网平台通行的功能域组织，覆盖设备接入、物模型、数据、规则告警、可视化、开放能力、系统管理、安全运维等模块。状态标记基于当前代码（`server/`、`admin/`、[[UbiBot开放平台硬件通信协议]]）的实际实现情况，作为功能规划与查漏补缺的依据，而非全部已完成。

图例：✅ 已实现　🚧 部分实现　⏳ 规划中/未实现　💡 可选（商业/进阶功能，开源版可不做）

## 1. 系统管理（后台基座）

| 功能点 | 说明 | 状态 |
| --- | --- | --- |
| 管理员账号管理 | 增删改查、启停用、重置密码 | ✅ |
| 角色管理 | 角色增删改查、启停用 | ✅ |
| 菜单/权限管理（RBAC） | 菜单树、按钮/API级权限、角色-菜单、用户-角色绑定 | ✅ |
| 客户（终端用户）账号管理 | 与管理员账号体系分离的C端/客户账号 | ✅ |
| 数据字典 | 字典类型+字典项，供下拉选项等复用 | ✅ |
| 参数配置 | 键值对系统参数 | ✅ |
| 文件管理 | 上传、目录树、去重（SHA哈希）、公开分享链接 | ✅ |
| 操作审计日志 | 记录操作人/模块/动作/目标对象/IP/UA | ✅ |
| 站内消息/通知中心 | 消息分类、系统通知/用户私信、已读状态、WebSocket实时推送 | ✅ |
| 定时任务调度 | Cron任务、失败重试、分布式锁、执行日志、手动触发 | ✅ |
| 登录安全 | 验证码、登录失败锁定、强制下线、多端登录管理 | ⏳ |
| 组织/部门管理 | 管理员按部门分组管理（若需要多级组织架构） | ⏳ 💡 |
| 系统级操作日志脱敏 | 审计日志中敏感字段（密码/密钥）自动脱敏 | ⏳ |

## 3. 设备接入与通信

| 功能点 | 说明 | 状态 |
| --- | --- | --- |
| HTTP设备协议 | 时间同步、HMAC签名激活、token鉴权与续期 | ✅ |
| 防重放机制 | nonce一次性 + 时间窗口 + 建议的ts单调递增 | ✅ |
| 数据上报接口 | 批量时间点+多传感器读数上报 | ✅ |
| 配置轮询（GET） | 免上报数据、快速拉取待下发指令 | ✅（协议已定义） |
| 指令下发通道 | 借助响应体携带cfg/cmd，无需设备可被动推送 | ✅ |
| MQTT接入 | 常见IoT平台标配的长连接接入方式，适合高频/低延迟场景 | ⏳ |
| CoAP/LwM2M接入 | 面向超低功耗蜂窝/NB-IoT设备的轻量协议 | ⏳ 💡 |
| 设备网关/子设备 | 网关代理多个子设备接入（如Zigbee子设备通过网关上云） | ⏳ 💡 |
| 在线状态判定 | 基于上次上报时间+上报间隔推算在线/离线，前端展示 | ⏳ |
| 连接数/吞吐量监控 | 接入层的实时连接数、QPS等指标 | ⏳ |

---

以上"⏳"项是典型物联网开放平台（如ThingsBoard、阿里云IoT、涂鸦IoT等）的常见能力集合，建议后续按"设备接入→数据→规则告警→可视化→开放能力"的顺序排优先级：先把历史数据存储查询、在线状态判定、告警规则这几项补齐，能覆盖大部分IoT场景的核心诉求；OTA、多租户计费、场景联动等可视产品定位再决定是否投入。
