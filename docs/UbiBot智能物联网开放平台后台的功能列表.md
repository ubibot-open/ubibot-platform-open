# UbiBot智能物联网开放平台后台的功能列表

旧脚手架（`platform/ubibot-platform-open`）与现有的内存态协议参考服务端（`server/internal`）都不再作为基础，整个工程（含设备协议服务端）全新构建。本清单不再标注"是否已实现"，改为标注**建设优先级**，作为从零搭建时的排期依据。[[UbiBot开放平台硬件通信协议]] 作为设备侧通信的设计契约保留，新服务端按它实现。

优先级含义：
- **P0（一期/MVP闭环）**：打通"设备上报数据→后台能看到→后台能下发指令并生效"这条主链路的必要条件，缺一个都跑不起来。
- **P1（二期/常用能力）**：绝大多数IoT场景的刚需，MVP跑起来之后紧接着要补的。
- **P2（三期/运营与体验）**：让平台好用、好运营，但不影响核心链路能否工作。
- **P3 💡（四期/可选进阶）**：视产品定位决定是否要做，很多可以留给商业版或后续迭代。

## 推荐建设路线

**一期（P0，MVP闭环）**：重写设备协议服务端（鉴权、防重放、did-token绑定、数据上报、去重）+ 配套持久化存储（设备身份/token/历史数据）+ 设备端C SDK + 最小后台（登录、设备列表/详情、手动下发一条指令并看到ack）。这一期跑通之后，"设备能连上、后台能看、能下发指令"这条主线才算真正存在。

**二期（P1，补齐常用能力）**：探头自定义数据读取（set_probe落地+nak反馈）、历史数据查询、在线状态判定、阈值/离线告警、RBAC与操作审计、指令管理页面、请求限流。

**三期（P2，运营体验）**：仪表盘/报表、OTA、消息中心与定时任务、开放API、文件/字典/参数等系统管理配套、系统监控。

**四期（P3，可选进阶）**：多租户计费、场景联动、CoAP/网关子设备、第三方集成、灾备多活——视产品定位再决定是否投入。
