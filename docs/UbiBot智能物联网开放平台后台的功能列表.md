# UbiBot智能物联网开放平台后台的功能列表

本清单按物联网平台通行的功能域组织，覆盖设备接入、物模型、数据、规则告警、可视化、开放能力、系统管理、安全运维等模块，可作为功能规划与查漏补缺的依据。

**关于状态标注的更正**：上一版误将 `platform/ubibot-platform-open`（一个包含完整RBAC后台的旧脚手架）当作本项目现状来源。但项目实际目录是 `E:\ubibot-open\ubibot-platform-open`，其 git 历史（`a28f70e` "移除原有旧版平台代码与配置文件"）显示那套RBAC后台、规则引擎、MQTT/CoAP、Home Assistant集成等已被你主动删除，当前代码只保留了：`server/internal`（设备协议参考服务端，纯内存态，无数据库）+ `csrc`（设备端C SDK）。本版状态已完全按这个真实范围重新核对，因此绝大部分条目从"已实现"改回"未实现"。

图例：✅ 已实现　🚧 部分实现　⏳ 规划中/未实现　💡 可选（商业/进阶功能，开源版可不做）

---

**现状总结**：项目目前是一个打磨得比较扎实的"设备接入协议参考实现"（HTTP鉴权、防重放、去重、指令通道的核心机制都对，且有测试覆盖），但完全没有后台管理系统——没有数据库、没有管理界面、没有除demo设备外的任何设备注册方式、也没有任何接口能真正触发一次指令下发。

建议的下一步优先级：① 给设备/token/telemetry接入持久化存储（当前重启即丢数据，是最大短板）；② 补一个最小可用的设备管理接口（注册设备、下发cfg/cmd），把已经写好的`QueueCommand`/`SetConfig`真正接上API；③ 再谈历史数据查询、在线状态判定、告警——这几项是把"协议"变成"可用平台"的关键缺口；RBAC后台、OTA、多租户计费、规则引擎这些可以等核心链路打通后再规划。
